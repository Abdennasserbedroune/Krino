# Vercel Deployment Checklist ✅

Use this checklist before deploying to Vercel to ensure everything is configured correctly.

## Pre-Deployment Checklist

### 🔒 Security Check
- [ ] No API keys or secrets hardcoded in source code
- [ ] `.env`, `.env.local` added to `.gitignore`
- [ ] All sensitive files listed in `.gitignore`
- [ ] `.env.example` files contain only placeholder values
- [ ] Real secrets prepared for Vercel environment variables

### 📦 Configuration Files
- [ ] `vercel.json` created at project root
- [ ] `frontend/next.config.mjs` configured with Vercel settings
- [ ] `frontend/package.json` has correct build scripts
- [ ] Root directory set to `frontend` in Vercel project settings

### 🌍 Environment Variables Ready
Required for Production:
- [ ] `NEXT_PUBLIC_SUPABASE_URL` - Supabase project URL (from Supabase Dashboard → API)
- [ ] `NEXT_PUBLIC_SUPABASE_ANON_KEY` - Supabase anonymous key (from Supabase Dashboard → API)
- [ ] `GROQ_API_KEY` - Get from https://console.groq.com/keys
- [ ] `NEXTAUTH_URL` - Vercel deployment URL (e.g., https://your-app.vercel.app)
- [ ] `NEXTAUTH_SECRET` - Generated with `openssl rand -base64 32`

Optional (legacy):
- [ ] `NEXT_PUBLIC_API_BASE_URL` - Backend API URL (leave empty for same-domain)

Optional (if using):
- [ ] `GOOGLE_CLIENT_ID` - Google OAuth
- [ ] `GOOGLE_CLIENT_SECRET` - Google OAuth
- [ ] `EMAIL_SERVER_HOST` - Email provider
- [ ] `EMAIL_SERVER_PORT`
- [ ] `EMAIL_SERVER_USER`
- [ ] `EMAIL_SERVER_PASSWORD`
- [ ] `EMAIL_FROM`

### 🗄️ Database & Storage Setup (Supabase)
- [ ] Supabase project created
- [ ] `cvs` table created (see SUPABASE_SETUP.md for SQL)
- [ ] `cvs` storage bucket created
- [ ] Row Level Security (RLS) policies configured for table
- [ ] Storage RLS policies configured for bucket
- [ ] Authentication enabled (Email provider)
- [ ] Site URL and Redirect URLs configured in Auth settings

### 🔌 Backend API
- [ ] Backend deployed to separate platform
- [ ] Backend API URL is accessible
- [ ] Backend CORS allows Vercel domain
- [ ] Backend environment variables set:
  - [ ] `DATABASE_URL`
  - [ ] `GROQ_API_KEY`
  - [ ] `CORS_ORIGIN` includes Vercel URL
  - [ ] `JWT_SECRET`

### 📝 Code Quality
- [ ] Run `cd frontend && npm run lint` (no errors)
- [ ] Run `cd frontend && npm run build` (successful)
- [ ] All imports resolve correctly
- [ ] TypeScript compilation successful

### 🔍 Code Review
- [ ] All hardcoded `localhost` URLs replaced with env vars
- [ ] API calls use `process.env.NEXT_PUBLIC_API_BASE_URL`
- [ ] No console.log statements with sensitive data
- [ ] Error handling in place for API failures

## Deployment Steps

### 1. Initial Deployment
```bash
# Push code to GitHub
git add .
git commit -m "Configure for Vercel deployment"
git push origin main

# Option A: Deploy via Vercel Dashboard
# 1. Go to vercel.com/new
# 2. Import GitHub repository
# 3. Set Root Directory to "frontend"
# 4. Click Deploy

# Option B: Deploy via CLI
npm i -g vercel
vercel --cwd frontend
```

### 2. Configure Environment Variables
1. Go to Vercel Dashboard → Your Project
2. Settings → Environment Variables
3. Add all required variables for:
   - Production
   - Preview
   - Development
4. Click "Save"

### 3. Redeploy
```bash
# After adding environment variables, redeploy
vercel --cwd frontend --prod

# Or trigger redeploy from Vercel Dashboard:
# Deployments → Latest Deployment → ⋯ → Redeploy
```

- [ ] API calls reach backend successfully

- [ ] Database connections working

## Troubleshooting

### Build Fails
1. Check build logs in Vercel Dashboard
2. Verify `package.json` scripts
3. Ensure all dependencies are in `dependencies` (not `devDependencies`)
4. Check for TypeScript errors

### Environment Variables Not Working
1. Ensure variables are set for correct environment
2. Client-side variables must start with `NEXT_PUBLIC_`
3. Redeploy after adding new variables
4. Check variable names match exactly (case-sensitive)

### API Calls Failing
1. Verify `NEXT_PUBLIC_API_BASE_URL` is set
2. Check backend CORS allows Vercel URL
3. Ensure backend is deployed and accessible
4. Check network tab in browser dev tools

### Database Connection Issues
1. Verify `DATABASE_URL` format is correct
2. Check database allows connections from Vercel IPs
3. Test database connection from backend
4. Run migrations if needed

## Rollback Plan

If deployment fails:
```bash
# Option 1: Rollback via CLI
vercel rollback [deployment-url]

# Option 2: Rollback via Dashboard
# Deployments → Previous Working Deployment → ⋯ → Promote to Production
```
