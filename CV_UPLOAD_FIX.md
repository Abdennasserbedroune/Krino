# CV Upload 500 Error Fix

## Problem
The CV upload endpoint (`/api/v1/cv/upload`) is returning a 500 Internal Server Error on Vercel deployment.

## Root Cause Analysis
The issue is caused by CommonJS/ESM module compatibility problems with the file parsing libraries (`pdf-parse` and `mammoth`) in Vercel's serverless environment.

### Specific Issues:
1. **Top-level imports of CommonJS modules**: Both `pdf-parse` and `mammoth` are CommonJS modules that were being imported at the top level in an ESM project
2. **Missing runtime specification**: Next.js API routes default to Edge runtime on Vercel, which doesn't support Node.js-specific APIs like `Buffer`
3. **Module resolution errors**: The static imports fail in the serverless environment

## Changes Made

### File: `frontend/app/api/v1/cv/upload/route.ts`

#### 1. Removed top-level imports (Lines 1-4)
**Before:**
```typescript
import { NextRequest, NextResponse } from "next/server";
import { createClient } from "@/lib/supabase/server";
import pdfParse from "pdf-parse";
import mammoth from "mammoth";
```

**After:**
```typescript
import { NextRequest, NextResponse } from "next/server";
import { createClient } from "@/lib/supabase/server";

export const runtime = 'nodejs';
```

#### 2. Added Node.js runtime specification (Line 4)
Added `export const runtime = 'nodejs'` to ensure the route runs in Node.js environment which supports Buffer and file operations.

#### 3. Changed to dynamic imports for file parsing (Lines 67-86)
**Before:**
```typescript
if (ext === "pdf") {
    const pdfData = await pdfParse(buffer);
    rawText = pdfData.text || "";
    pageCount = pdfData.numpages || 1;
    // ...
} else if (ext === "docx" || ext === "doc") {
    const result = await mammoth.extractRawText({ buffer });
    rawText = result.value || "";
    // ...
}
```

**After:**
```typescript
if (ext === "pdf") {
    console.log("Parsing PDF...");
    const pdfParse = (await import("pdf-parse")).default;
    const pdfData = await pdfParse(buffer);
    rawText = pdfData.text || "";
    pageCount = pdfData.numpages || 1;
    // ...
} else if (ext === "docx" || ext === "doc") {
    console.log("Parsing DOCX/DOC...");
    const mammoth = await import("mammoth");
    const result = await mammoth.extractRawText({ buffer });
    rawText = result.value || "";
    // ...
}
```

#### 4. Enhanced error logging (Lines 134-147)
Added detailed error messages and stack traces to help diagnose issues in production:
```typescript
} catch (error: any) {
    console.error("Upload handler failed:", error);
    console.error("Error stack:", error?.stack);
    return NextResponse.json(
        { detail: "Internal Server Error", message: error?.message || "Unknown error" },
        { status: 500 }
    );
}
```

## Required Configuration in Vercel

### Environment Variables
Make sure these are set in Vercel Dashboard → Settings → Environment Variables:

1. **NEXT_PUBLIC_SUPABASE_URL** - Your Supabase project URL
   - Example: `https://your-project.supabase.co`

2. **NEXT_PUBLIC_SUPABASE_ANON_KEY** - Your Supabase anon/public key
   - Get from: Supabase Dashboard → Project Settings → API

3. **DATABASE_URL** (for backend, if applicable)
   - Your PostgreSQL connection string

4. **NEXTAUTH_SECRET** (for authentication)
   - Generate with: `openssl rand -base64 32`

### Supabase Setup
Ensure the following exists in your Supabase project:

1. **Storage Bucket**: A bucket named `cvs` must exist
   - Create in: Supabase Dashboard → Storage → New bucket
   - Name: `cvs`
   - Make it public or configure appropriate RLS policies

2. **Database Table**: A `cvs` table must exist with the following schema:
   ```sql
   CREATE TABLE cvs (
       id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
       user_id UUID REFERENCES auth.users(id) NOT NULL,
       original_filename TEXT NOT NULL,
       file_path TEXT NOT NULL,
       file_type TEXT NOT NULL,
       file_size BIGINT NOT NULL,
       extracted_cv JSONB,
       analysis_result JSONB,
       score INTEGER,
       structured_data JSONB,
       created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
   );
   ```

3. **RLS Policies**: Row Level Security policies should allow users to:
   - Read their own CVs
   - Insert their own CVs
   - Delete their own CVs

## Common Issues and Solutions

### Issue: "Failed to upload file to storage"
**Cause**: Supabase storage bucket doesn't exist or permissions issue
**Solution**: Create the `cvs` bucket in Supabase Dashboard

### Issue: "Failed to save CV record"
**Cause**: Database table doesn't exist or schema mismatch
**Solution**: Run the SQL migration to create the `cvs` table

### Issue: "Unauthorized" (401)
**Cause**: User not authenticated or session expired
**Solution**: Ensure user is signed in and Supabase auth is working

### Issue: Timeout on large files
**Cause**: Vercel serverless functions have a 10s timeout limit
**Solution**: Keep files under 5MB (already enforced in code)
